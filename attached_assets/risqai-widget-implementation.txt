RisqAi Widget Implementation Guidelines
---------------------------------------
Overview:
Widgets are modular UI components that follow a consistent tile-based design pattern,
providing a cohesive and unified visual experience across the RisqAi platform.

Core Structure:
1. Container
   - Gradient background (from-[#300A45]/80 to-black/80)
   - Subtle border (border-[#BF00FF]/20)
   - Border radius (rounded-xl)
   - Drop shadow (shadow-lg shadow-[#BF00FF]/5)
   - Padding (p-6)
   - Flexbox layout (flex flex-col)

2. Header Section
   - Visual identifier (icon, example component, or graphic)
   - Title (text-xl font-bold mb-2)
   - Description (text-sm text-muted-foreground mb-4)
   
3. Content Section
   - Main content area
   - Flexible layout based on content needs
   - Consistent spacing (gap-4 or gap-6)
   
4. Footer/Action Section
   - Positioned at bottom (mt-auto)
   - Contains action buttons or download options
   - Consistent grid layout (grid grid-cols-3 gap-2)
   - Explanatory text (text-xs text-muted-foreground text-center)

Implementation:
```jsx
<motion.div
  initial={{ opacity: 0, y: 20 }}
  whileInView={{ opacity: 1, y: 0 }}
  transition={{ duration: 0.5, delay: 0.3 }}
  viewport={{ once: true }}
  className="bg-gradient-to-b from-[#300A45]/80 to-black/80 backdrop-blur-sm 
             border border-[#BF00FF]/20 rounded-xl p-6 flex flex-col 
             shadow-lg shadow-[#BF00FF]/5"
>
  {/* Header */}
  <div className="mb-4">
    <div className="mb-4">
      {/* Visual identifier goes here */}
    </div>
    <h3 className="text-xl font-bold mb-2">Widget Title</h3>
    <p className="text-sm text-muted-foreground mb-4">
      Widget description text goes here.
    </p>
  </div>
  
  {/* Content */}
  <div className="flex-1">
    {/* Main content goes here */}
  </div>
  
  {/* Footer/Actions */}
  <div className="mt-auto flex flex-col gap-2">
    <div className="grid grid-cols-3 gap-2">
      {/* Action buttons go here */}
    </div>
    <span className="text-xs text-muted-foreground text-center">
      Explanatory text
    </span>
  </div>
</motion.div>
```

Animations:
- Each widget uses framer-motion for entry animations
- Staggered delays (0.1s between each widget) for visual hierarchy
- Only animate once when entering viewport

Responsiveness:
- Widgets adjust from 1 column (mobile) to 2 columns (tablet) to 4 columns (desktop)
- Content maintains readability at all sizes
- Action buttons remain accessible with minimum 44px touch targets

Accessibility:
- Maintain contrast ratio of at least 4.5:1 for text content
- Interactive elements should have visible focus states
- Ensure all interactive elements have appropriate ARIA attributes
- Use semantic HTML where possible

Variations:
1. Standard Widget - As described above
2. Interactive Widget - Adds hover effects and clickable regions
3. Metric Widget - Emphasizes data visualization over textual content
4. Expandable Widget - Can toggle between compact and expanded states
